# Emperical Update System

Locked: May 30, 2026

## Homepage Rule
The homepage is a manual editorial edition, not a live daily feed. The visible current-edition date must remain fixed until the site owner intentionally updates the homepage again.

## Archive Rule
Before publishing a new homepage edition, move the existing homepage into the Archive as the next edition entry.

## Story Rule
Each story card should use this structure:

1. Category
2. Summary
3. Why It Matters
4. Ripple Effect
5. Source

Summaries must be original, short, and clear. Sources must stay visible. Duplicate stories should be avoided.


## Story Card Visual Rule
Story field labels should be visually separated so the card does not read like one large block. Use subtle badge styling for Category, Summary, Why It Matters, Ripple Effect, and Source while preserving the black/gold Emperical identity.

## UI Rule
Preserve the black/gold Emperical identity, homepage layout, archive layout, navigation, and card style unless a UI change is specifically requested.

## SHA-256 Rule
Use SHA-256 seals for milestone/release packages. Skip hash seals for small UI/text tweaks unless requested.

## Story Card Border Rule
The story title remains the outer card headline. The inner story panel must visibly wrap only the structured story fields: Category, Summary, Why It Matters, Ripple Effect, and Source. Keep the inner border strong enough to separate the fields from the headline so each card reads in layers instead of one solid text block.

## Story Card Headline Rule
Story card headlines, such as “Mystery Rescuer Saves Cyclist With CPR,” must remain outside the inner story panel and be centered at the top of each card. The inner panel begins below the centered headline and contains Category, Summary, Why It Matters, Ripple Effect, and Source.
